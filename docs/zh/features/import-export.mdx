---

Banana Slides 使用 Markdown 文件（`.md` / `.txt`）導入導出大綱和頁面描述，格式可讀性強，可用任意文本編輯器編輯。

## 格式結構

```markdown
# 項目標題

> 生成時間: 2025/1/1 12:00:00

---

## 第 1 頁: 封面

> 章節: 引言

**大綱要點：**
- 公司名稱和 Logo
- 演示日期

**頁面描述：**
簡潔的封面頁，公司 Logo 居中，標題使用白色粗體字，深藍漸變背景。

---

## 第 2 頁: 市場概況

> 章節: 分析

**大綱要點：**
- 行业增長趨勢
- 主要市場細分

**頁面描述：**
數據驅動的頁面，展示過去 5 年市場增長柱狀圖，關鍵數據用高亮卡片突出。

---
```

## 字段說明

| 字段 | 語法 | 必填 |
|------|------|------|
| 頁面標題 | `## 第 N 頁: 標題` | 是 |
| 章節 | `> 章節: 名稱` | 否 |
| 大綱要點 | `**大綱要點：**` 後跟 `- 要點` 行 | 否 |
| 頁面描述 | `**頁面描述：**` 後跟自由文本 | 否 |

頁面之間用 `---` 分隔。

## 導出選項

可從大綱編輯器和詳細編輯器導出：

| 來源 | 內容 |
|------|------|
| 大綱編輯器 | 僅大綱要點 |
| 詳細編輯器 | 大綱要點 + 頁面描述 |

## 導入行為

- 導入的頁面會**追加**到現有項目，不會替換已有頁面。
- `**大綱要點：**` 和 `**頁面描述：**` 標記可省略，省略時列表行（`- ...`）自動識別為要點。
- 導入內容中的 HTML 標簽會被自動清除。
- 支持 `.md` 和 `.txt` 文件。

## 最簡示例

一個有效的導入文件可以很簡單：

```markdown
## 第 1 頁: 開場介紹
- 歡迎與議程
- 演講者介紹

## 第 2 頁: 總結
- 核心要點回顧
```

## 用 AI 生成

將下面的提示詞複制到任意 AI 助手（ChatGPT、Claude、Kimi 等），替換占位符，輸出結果可直接導入 Banana Slides。

<Tip>
  如果只需要大綱（不含描述），從提示詞中刪除 `**頁面描述：**` 相關要求即可。
</Tip>

<Accordion title="AI 提示詞模板">

```text
請按以下 Markdown 格式生成一份演示文稿大綱。

主題：[你的主題]
頁數：[數量，如 8]
語言：[中文 / 英文]
風格備注：[如"專業數據風"或"活泼插畫風"]

格式要求：
1. 每頁以 "## 第 N 頁: 標題" 開頭
2. 用 "> 章節: 章節名" 對頁面分組
3. 在 "**大綱要點：**" 下用 "- 要點" 列出 2-4 個要點
4. 在 "**頁面描述：**" 下用 1-2 句話描述該頁的視覺布局、配色、圖表或畫面
5. 頁面之間用 "---" 分隔
6. 第一頁為封面，最後一頁為結束/致謝頁

只輸出 Markdown，不要額外解釋。
```
